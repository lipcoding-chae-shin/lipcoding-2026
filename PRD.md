## 1. 한 줄 요약
> 여러 채널 알림에 파묻힌 바쁜 직장인 "지우"가, Gmail·Slack 등에서 메시지가 쏟아져 **"지금 당장 뭘 해야 하는지"**를 놓치는데, 이 앱이 **Copilot SDK 트리아지 에이전트 + Azure OpenAI**로 통합 피드를 요약·우선순위화하고 '지금 할 일'을 골라준다.

`Gmail·Slack을 한 피드로 모아, AI가 '지금 할 일 Top N'을 골라주는 트리아지 인박스`

## 2. 타깃 사용자 & 문제 (생산성·문제 적합성 18%)
- **타깃 사용자(대표 페르소나 1명)**: "지우" — Gmail·Slack·기타 알림을 하루 종일 받는 바쁜 직장인. (제품은 범용이나 데모는 이 1명으로 시연)
- **구체적 고통**: 메시지가 여러 앱에 흩어져 쏟아져, 정작 **지금 응답/처리해야 할 항목**을 놓치고 컨텍스트 전환 비용이 큼.
- **이 앱이 주는 실질 이익(측정 가능)**: 흩어진 알림을 1개 피드로 통합 + AI가 **'지금 할 일 Top N'** 선정 → "무엇부터 할지" 판단 시간을 분 단위에서 초 단위로 단축.
- **범위 밖(안 만들 것)**: 실제 메일 전송, Slack 실연동, 다중 사용자/회원가입, 모바일 네이티브, 알림 푸시. ← 4시간 사수용.

## 3. 핵심 가치 흐름 (수직 슬라이스)
> 가장 먼저 끝까지 완성할 **단 하나의 흐름**. 데모에서 이걸 보여준다.

1. 사용자가 **Gmail을 연결(읽기 전용 OAuth)** 하고, Slack/기타는 mock 시드로 채워 **통합 피드**를 본다 →
2. 앱이 **Copilot SDK 트리아지 에이전트**로 피드 항목을 요약·우선순위화하고 **'지금 할 일 Top N'**을 선정한다 (에이전트 루프 + 도구 호출, 스트리밍) →
3. **Azure OpenAI** 모델이 각 항목의 요약·중요도·추천 액션을 생성한다 →
4. 결과를 **피드 + '지금 할 일' 패널** UI로 보여주고, 각 항목에 **답장 초안 / 할일 생성 / 스누즈** 액션을 제공한다(전송·생성 전 사람 승인).

## 4. Copilot SDK 활용 (★ 25% — 가장 중요, 깊이로 평가)
> 얕은 단발 호출 금지. **앱의 심장**이 되게.

- 무엇에 쓰나: **에이전트 루프 + 도구 호출(tool-use)** 을 깊게. 피드 컨텍스트를 주입해 멀티스텝 reasoning + **스트리밍**으로 트리아지 과정을 노출.
- 도구(tool) 목록: `summarize_thread`, `prioritize`(중요도/긴급도 산정), `draft_reply`(답장 초안), `create_task`(할일 생성), `snooze`.
- 깊이 증거(채점관에게 보여줄 1가지): 에이전트가 여러 피드 항목을 순회하며 **요약→우선순위→추천 액션**을 도구 호출로 단계적으로 처리하고, 그 reasoning을 스트리밍으로 실시간 표시.
- 사용 패키지/엔드포인트: `@github/copilot` (github.com/github/copilot-sdk)

## 7. UX / 워크플로우 (12%)
- 진입 후 **첫 가치까지 클릭 수**: Gmail 연결 1클릭(또는 mock 폴백 0클릭) → 피드+'지금 할 일'이 바로 뜸.
- AI가 어떻게 **자연스럽게** 녹는가: 모델을 전면에 노출하지 않고, 피드 우측 **'지금 할 일' 패널**과 항목별 추천 액션으로 녹임.
- **사용자 주도권**: 답장 초안은 편집 가능, 전송·할일 생성·스누즈는 **사용자 승인 후** 적용. 모든 액션 취소 가능.
