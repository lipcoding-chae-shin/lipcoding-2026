## 1. 한 줄 요약
> 여러 채널에서 정보가 쏟아지는 바쁜 직장인 "지우"가 **"지금 뭘 해야 하는지"**를 놓치는데, 이 앱이 소스를 **구독**받아 **Copilot SDK로 요약·태깅**하고, **Task로 분류된 항목을 Todo 리스트로** 만들어준다.

`소스를 구독하면 → AI가 요약·태깅 → Task는 자동 Todo. 왼쪽 피드, 오른쪽 AI Todo 리스트.`

## 2. 타깃 사용자 & 문제 (생산성·문제 적합성 18%)
- **타깃 사용자(대표 페르소나 1명)**: "지우" — Gmail·GitHub·사내 뉴스 등 여러 채널의 정보를 하루 종일 받는 바쁜 직장인. (제품은 범용이나 데모는 이 1명으로 시연)
- **구체적 고통**: 정보가 여러 앱에 흩어져 쏟아져, 정작 **내가 행동해야 할 항목(Task)**을 단순 참고(Info)와 구분하지 못하고 놓침.
- **이 앱이 주는 실질 이익(측정 가능)**: 구독한 소스를 1개 피드로 통합 + AI가 **요약·태깅 후 Task만 Todo로 추출** → "무엇부터 할지" 판단·정리 시간을 대폭 단축.
- **범위 밖(안 만들 것)**: 실제 메일 전송, 다중 사용자/회원가입, 모바일 네이티브, 알림 푸시, Slack 실연동. ← 4시간 사수용.

## 3. 핵심 가치 흐름 (수직 슬라이스)
> 가장 먼저 끝까지 완성할 **단 하나의 흐름**. 데모에서 이걸 보여준다.

1. 사용자가 소스를 **구독**한다 — **Gmail(읽기전용 OAuth)**, **GitHub(GitHub MCP/도구)**, 사내 뉴스(mock 시드) →
2. 구독으로 들어온 인풋을 **MCP/도구**로 수집해 **왼쪽 피드**에 모은다 →
3. **Copilot SDK 에이전트**가 각 항목을 **요약**하고, 요약을 바탕으로 **태그(Task / Info 등)**를 붙인다 (도구 호출, 스트리밍) →
4. **Task** 태그가 붙은 항목을 **오른쪽 AI Todo 리스트**로 생성한다 (사용자 승인·편집 가능).

## 4. Copilot SDK 활용 (★ 25% — 가장 중요, 깊이로 평가)
> 얕은 단발 호출 금지. **앱의 심장**이 되게.

- 무엇에 쓰나: **에이전트 루프 + 도구 호출(tool-use) + MCP 연동**을 깊게. 구독 소스의 인풋을 컨텍스트로 주입해 **요약 → 태깅 → Todo 추출**을 멀티스텝으로 처리, **스트리밍**으로 과정 노출.
- 도구(tool) 목록: `fetch_subscription`(MCP/소스 인풋 수집), `summarize_item`(요약), `tag_item`(Task/Info 분류), `create_todo`(Task→Todo 생성).
- MCP: **GitHub MCP**로 GitHub 인풋(이슈/PR/알림)을 수집 → SDK 도구 호출의 입력으로 사용.
- 깊이 증거(채점관에게 보여줄 1가지): 에이전트가 구독 항목을 순회하며 **요약→태깅→Todo 생성**을 도구 호출로 단계적으로 처리하고, 그 reasoning을 스트리밍으로 실시간 표시.
- 사용 패키지/엔드포인트: `@github/copilot` (github.com/github/copilot-sdk)

## 7. UX / 워크플로우 (12%)
- **레이아웃**: 좌측 = 구독한 메시지 **피드**(소스 아이콘 + 요약 + 태그 배지), 우측 = **AI Todo 리스트**.
- 진입 후 **첫 가치까지 클릭 수**: 소스 구독 1클릭(또는 mock 폴백 0클릭) → 피드+Todo가 바로 뜸.
- AI가 어떻게 **자연스럽게** 녹는가: 모델을 전면에 노출하지 않고, 피드 항목의 요약·태그 배지와 우측 Todo로 녹임.
- **사용자 주도권**: AI가 만든 Todo는 편집·삭제·완료 가능, 태그 수정 가능. 모든 액션 취소 가능.

## 8. 책임 있는 AI · 보안 (6%)
- **위험 작업 전 사람 확인 게이트**: AI가 Task→Todo를 **제안**하면 사용자가 승인/편집 후 확정. 자동 삭제·외부 전송 없음.
- 프라이버시: Gmail 읽기 전용(`gmail.readonly`), 본문은 요약·태깅에 필요한 최소만 모델에 전달, 영구 저장 최소화.
- 환각 완화: 각 요약/태그/Todo에 **원본 피드 항목 출처 링크** 표시, 불확실 시 '확인 필요' 라벨.
- 시크릿 관리: `.env`(로컬) + Azure Key Vault/App Settings(배포). 코드 미노출, `.env.example` 제공.

## 9. 혁신성 (5%)
- 단순 통합 인박스가 아니라, **구독 소스를 AI가 요약·태깅해 Task만 자동으로 Todo로 승격** — 정보 소비를 행동(Todo)으로 잇는 트리아지.
